ORES Deployment to Equinor Azure Cloud
=======================================

Recommended: Azure App Service (Linux, B1 tier)
------------------------------------------------
Simplest "small always-on webserver" option. Natively supports Python,
custom domains, integrates with Equinor Azure AD tenant.

1. Create a Dockerfile
-----------------------
  FROM python:3.12-slim
  WORKDIR /app
  COPY requirements.txt .
  RUN pip install --no-cache-dir -r requirements.txt
  COPY app/ ./app/
  EXPOSE 8000
  CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "2"]

2. Create Azure Container Registry + App Service
-------------------------------------------------

Request: Contributor role on subscription S449-OSDU-Test (or a new resource group rg-ores created for me on that subscription with Contributor access).
Purpose: Deploy a small internal web app (ORES - OSDU admin UI) as an Azure App Service container.
Resources needed: Resource Group, Azure Container Registry (Basic), App Service Plan (B1 Linux)
User: MAAP@equinor.com (object ID: b97121e4-2d6d-4965-bf00-4795c65497f8)


  # Variables
  RG="rg-ores"
  ACR="acroresprod"
  APP="app-ores"
  PLAN="plan-ores"
  LOCATION="northeurope"          # Equinor primary region

  # Resource group
  az group create -n $RG -l $LOCATION

  # Container registry
  az acr create -n $ACR -g $RG --sku Basic --admin-enabled true

  # Build & push image
  az acr build -r $ACR -t ores:latest .

  # App Service plan (B1 = always-on, ~$13/mo)
  az appservice plan create -n $PLAN -g $RG --is-linux --sku B1

  # Create web app from the ACR image
  az webapp create -n $APP -g $RG -p $PLAN \
    --docker-registry-server-url https://$ACR.azurecr.io \
    -i $ACR.azurecr.io/ores:latest

3. Configure environment variables
-----------------------------------
  az webapp config appsettings set -n $APP -g $RG --settings \
    SECRET_KEY="$(openssl rand -hex 32)" \
    DEBUG="False" \
    LOG_LEVEL="INFO" \
    OSDU_BASE_URL="equinorswedev.energy.azure.com" \
    DATA_PARTITION_ID="dev" \
    AZURE_TENANT_ID="3aa4a235-b6e2-48d5-9195-7fcf05b459b0" \
    AZURE_CLIENT_ID="7a414874-4b27-4378-b34f-bc9e5a5faa4f" \
    AZURE_SCOPE="7daee810-3f78-40c4-84c2-7a199428de18/.default openid offline_access" \
    DEFAULT_LEGAL_TAG="dev-equinor-private-default" \
    DEFAULT_COUNTRIES="NO" \
    DEFAULT_OWNERS="data.default.owners@dev.dataservices.energy" \
    DEFAULT_VIEWERS="data.default.viewers@dev.dataservices.energy" \
    APP_KEY="test-app" \
    WEBSITES_PORT="8000"

  NOTE: Do NOT set REFRESH_TOKEN in production. The app already supports
  per-user PKCE login (AUTH_MODE="per_user_pkce"), which is correct for
  multi-user deployment — each Equinor user authenticates with their own
  Azure AD identity.

4. Update Azure AD App Registration
------------------------------------
  In Azure Portal -> App Registrations, find client ID 7a414874-... and set:
  - Redirect URI:            https://app-ores.azurewebsites.net/auth/callback
  - Supported account types: Accounts in this organizational directory only (Equinor)
  - ID tokens:               Enabled

5. Enable HTTPS-only and always-on
------------------------------------
  az webapp update -n $APP -g $RG --https-only true
  az webapp config set -n $APP -g $RG --always-on true

  Also update session middleware in app/main.py line 62:
    https_only=os.getenv("DEBUG", "True").lower() != "true",  # True in prod

6. (Optional) Custom domain
-----------------------------
  az webapp config hostname add -n $APP -g $RG --hostname ores.equinor.com
  az webapp config ssl create -n $APP -g $RG --hostname ores.equinor.com

7. CI/CD with GitHub Actions
------------------------------
  # .github/workflows/deploy.yml
  name: Deploy ORES
  on:
    push:
      branches: [main]
  jobs:
    deploy:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
        - uses: azure/login@v2
          with:
            creds: ${{ secrets.AZURE_CREDENTIALS }}
        - run: az acr build -r acroresprod -t ores:latest .
        - run: az webapp restart -n app-ores -g rg-ores

Key architectural points for Equinor multi-user access
------------------------------------------------------
1. Authentication: Per-user PKCE auth is built in. When REFRESH_TOKEN is
   empty, each user goes through Azure AD login with their own Equinor
   identity — exactly what you want in production.

2. OSDU authorization: Each user's own access token is used for OSDU API
   calls, so OSDU role-based access control applies automatically.

3. Cost: App Service B1 (~$13/mo) is sufficient. Scale to S1 if you need
   staging slots or more CPU.

4. Networking: If the OSDU instance requires VNet access, place the App
   Service in a VNet with a service endpoint or use VNet integration.

Alternative options:
  - Azure Container Apps:    Serverless scaling, cheaper at low traffic (cold starts)
  - Azure Container Instance: Simplest container hosting (no built-in TLS)
  - Azure Kubernetes Service: Full control (overkill for single small app)

  App Service is the sweet spot for "small always-on webserver available
  to all Equinor users."
