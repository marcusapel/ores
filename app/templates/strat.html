Generate Horizons from Unit Boundaries

Derive HorizonInterpretation records from the age boundaries of units in this column. Each unique boundary age produces one horizon. Existing horizons are kept - only missing boundaries are generated.

Generate Units from Horizon Boundaries

Derive StratigraphicUnitInterpretation records from consecutive horizon boundaries. Each pair of adjacent horizons (sorted by age) produces one unit interval. Existing units are kept - only missing intervals are generated.

Import & Ingest Stratigraphic Columns

OpenWorks JSON SMDA API OSDU Bundle File Push to RDDMS

Upload an OpenWorks JSON export with {"StratColumn":{"Name":"..."}, "Units":[...]} structure.

What do these buttons do?
  • Convert to OSDU - Parses the uploaded OpenWorks JSON and converts it into OSDU WPC records (StratigraphicColumn + ColumnRankInterpretation + UnitInterpretation). Nothing is sent yet - preview the result first.
  • Send to OSDU - Writes each record to the OSDU Storage API in batches. Creates the stratigraphic column and all its components in the OSDU records catalog.
  • Download Bundle - Saves the OSDU WPC bundle as a .json file for inspection, version control, or later upload.

Fetch a stratigraphic column from the SMDA API Gateway and convert to OSDU format.

What do these buttons do?
  • Fetch & Convert - Calls the SMDA REST API to retrieve all stratigraphic units for the selected column, then converts them to OSDU WPC records. Preview the result below - nothing is written to OSDU yet.
  • Send to OSDU - Writes each record to the OSDU Storage API in batches. Creates the column, ranks, and units in the OSDU records catalog.
  • Download Bundle - Saves the OSDU WPC bundle as a .json file for inspection or later upload.
  • Batch Fetch & Ingest Selected - Fetches and sends to OSDU each selected column in sequence. Use Ctrl-click or Shift-click to select multiple columns in the list.
Push to Reservoir DDMS as RESQML

Converts the SMDA column to RESQML 2.0.1 objects (StratigraphicColumn, ColumnRankInterpretation, UnitInterpretation) and pushes them to the Reservoir DDMS dataspace. This is a separate path from OSDU ingestion - it stores the stratigraphic model in the RESQML-native reservoir data store.

Upload a pre-built OSDU WPC bundle JSON ({"records":[...]}) for direct ingestion. This is typically a file previously exported via "Download Bundle" from the other tabs, or hand-crafted.

What does this do?
  • Send to OSDU - Writes each record from the uploaded bundle to the OSDU Storage API in batches. Creates stratigraphic columns, ranks, and units in the OSDU records catalog.
