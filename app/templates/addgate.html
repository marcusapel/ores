Create Generic Record

  

    Create an OSDU record by choosing a type and filling in the data fields.
    Focused on SME-authored metadata — governance objects, assessments, and entity definitions
    that subject-matter experts create manually rather than application pipelines.
  


  
  

    
1. Record Type


    
    

      
      
      
      
      
      
      
      
      
      
      
      
    


    

      
      
      Full OSDU kind string. Edit to adjust version or use a custom schema.
      
        

      
      
      Leave blank for UUID-based auto ID, or enter like dev:master-data--Risk:MyRisk:1.

      Build the data block. Fields are pre-populated for the selected type —
      fill in values, remove what you don’t need, and add extras.

Activity & Template Model


    

      
ActivityTemplate
defines parameter slots

      →
      
Activity
fills slots with values

    


    
How it works

    
    
      
  1. Create a Template — define the reusable shape of a workflow: what inputs it needs (e.g. “Process”, “NumberOfRealizations”), what outputs it produces (e.g. “OutputVolumes”). Each slot has a title, role (input/output), and expected kind (string, integer, or DataObject reference).
    2. 
      
  2. Run the workflow in your tool (RMS, ERT, custom script).
    3. 
      
  3. Create an Activity — record the execution by referencing the template ID and filling each parameter slot with the actual value used (a string, a number, or an OSDU record ID for DataObject parameters).
    4. 
    


    
Template parameter slots

    

      
FieldMeaning

      
TitleSlot name — Activities must use the same title

      
IsInput / IsOutputMarks the slot’s role

      
DefaultParameterKindExpected value type: string, integer, DataObject

      
MinOccurs / MaxOccursCardinality (0=optional, 1=required)

    


    
Activity parameter values

    

      
FieldMeaning

      
TitleMust match a template slot title

      
ParameterKindIDRef-data link: ParameterKind:String, :Integer, :DataObject

      
ParameterRoleIDRef-data link: ParameterRole:Input or :Output

      
StringParameter / IntegerParameter / DataObjectParameterThe actual value in the typed field

    


    
Quick tips

    
    
      
  • Use the Template presets (Reservoir Simulation, Interpretation, QC) to scaffold common workflows instantly.
    • 
      
  • After creating a template, copy the returned ID into the Activity form’s Template ID field and click Load Slots to auto-populate parameters.
    • 
      
  • The Activity tab is for SME-authored provenance. Pipeline-generated activities (e.g. from ERT/FMU automation) can also be created here for testing.
    • 
      
  • For the full guide with worked examples (reservoir simulation, seismic interpretation, QC, FMU), see Activity & Provenance Guide.
